## Topic 2: Copying Pages and Records Into Language-Restricted Targets

Martin raised a concern about the already merged [copy patch #92580](https://review.typo3.org/c/Packages/TYPO3.CMS/+/92580). He asked whether copying a translated page tree into a parent page that itself has no translation would remove translations from the copied child pages. Eric explained that TYPO3 must distinguish between copying records and copying pages.

For records such as `tt_content` or `news`, the target page's available translations and the target `site configuration` matter. If the target page does not support the source record's language, the translation is discarded. For pages, however, the copied pages bring their own translations with them, and the check is based on the `site configuration`, not on whether the immediate parent page has a corresponding translation. André confirmed that this distinction is correct, so the team did not spend time reproducing the case manually.

## Topic 3: MM Relation Concept, Initiative Response, and DataHandler Refactoring

The team discussed Benni's [MM table patch #92859](https://review.typo3.org/c/Packages/TYPO3.CMS/+/92859) for improving bidirectional `MM` relations in TYPO3. André summarized that Benni had identified more than 25 related bugs around `MM` handling, Workspaces, Extbase, `allowLanguageSynchronization`, and `l10n_mode=exclude`. The proposed direction is to add `sys_language_uid` and workspace context columns to `MM` tables, while storing live default-language UIDs for both sides of the relation.

Eric and André read through the concept and discussed its implications. The core idea is that the local table's language awareness determines the `MM` storage behavior, while the foreign table's language awareness is handled at overlay or display level. For language-aware local tables, each translation can own its own relation unless the field is shared through `l10n_mode=exclude`. For non-language-aware local tables, the relation remains language-neutral.

The team compared the proposal with the initiative's long-term idea of making both sides of an `MM` relation language-consistent. Eric noted that the current concept still routes storage through default-language UIDs, but that this could be a clean and uniform foundation for a future step toward fully language-aware relations.

Eric drafted and posted a Gerrit comment on behalf of the initiative in the related [patch #67172](https://review.typo3.org/c/Packages/TYPO3.CMS/+/67172). The comment stated that the team had originally considered language consistency on both the local and foreign sides, but that preparatory work is still needed before this can become possible. The initiative therefore sees Benni's approach as a promising, well-structured basis for future `MM` table optimizations.

André mentioned that Benni had merged [patch #92881](https://review.typo3.org/c/Packages/TYPO3.CMS/+/92881), a refactoring that separates `localizeRecord` and `copyRecord` logic in the `DataHandler`. Eric and André considered this a useful cleanup because it makes the code paths easier to understand and review.

The team also noted that the remaining `-1`-related work still depends on continued test coverage and careful review. Eric observed that if Benni's `MM` relation work enters TYPO3 v14, it could reduce several current relation bugs and provide a better base for later migration work.

## Topic 4: Camino Language Navigation, Changelog Handling, and Regional Sites

Eric presented the state of the `theme-camino` [language navigation patch #92909](https://review.typo3.org/c/Packages/TYPO3.CMS/+/92909). The patch had originally included a feature changelog entry, but because Camino had already been introduced in TYPO3 v14.1 and the navigation work would only become available later, Garvin argued that the changelog entry would be misleading.

Eric, Martin, and André agreed that the patch should remain a task without a standalone changelog entry. They considered documenting the language navigation in the `theme-camino` documentation instead, where a version-specific note for TYPO3 v14.2 would be more accurate than altering the existing v14.1 feature changelog.

While testing the Camino language navigation, Martin noticed visual differences between the language dropdown and the main navigation dropdown. The team also discussed how unavailable languages should be represented. Martin strongly argued that languages available elsewhere on the site should not disappear from the language navigation merely because a specific page is not translated. In his view, they should remain visible and disabled or lead to an explanatory page, so visitors understand that the language exists but the current page is not translated.

Eric then described a client case where the navigation mixes translated language variants with separate regional root sites, such as national sites for Belgium and the Netherlands. Martin stressed that such navigation must be designed from the visitor's perspective rather than from internal company structures. He argued that if visitors cannot understand whether they are switching language, region, or site context, the navigation produces usability problems even if it reflects the organization's internal logic.
