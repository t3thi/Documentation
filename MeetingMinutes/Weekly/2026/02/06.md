## Topic 2: Free-Mode Content Rendering Bugfix

Eric presented the finalized bugfix *“Show free-mode content if default column is empty”*. He explained that the issue in TYPO3 v14 is caused by *Default Language Binding* now being the default behavior in the Content Layout Module. This change removed a distinction that previously existed at the Fluid level and led to translated content not being rendered if the default language column was empty.

The implemented solution introduces a fallback case that explicitly renders translated content when no default-language content exists. Eric described the additional functional test and the custom YAML scenario that was required to accurately represent this edge case, where language `0` has no content while another language does. André confirmed that the patch works correctly in practice and does not introduce regressions.

The corresponding patch can be found at:  
https://review.typo3.org/c/Packages/TYPO3.CMS/+/92585

## Topic 3: Fluid Template Structure and Readability

A detailed discussion followed about the internal structure of the Fluid template and the readability of the fallback logic. André raised concerns about duplicated logic and repeated reassignment of the `column` variable, which makes the code harder to follow. Eric explained that the current approach deliberately avoids a larger refactoring in order to keep the change minimal and low-risk.

It was agreed that, while the structure is not ideal from a readability perspective, it is acceptable given the limited scope of the bugfix and the potential risks of deeper structural changes. As a small improvement, variable naming in the fallback block was aligned to reduce unnecessary indirection.

## Topic 4: `EXT:dbdoctor` Orphan Translation Record Check

Eric introduced a new `EXT:dbdoctor` check that identifies orphaned translation records. He reported that the rule already finds relevant issues in real-world installations and that the functional tests are passing. The intention is to open a pull request as a discussion baseline, explicitly marked as WIP to avoid premature merging.

The discussion focused on execution order within `EXT:dbdoctor`, as earlier rules may already modify or remove records that influence later checks. André emphasized that the orphan-translation check should run relatively early to avoid misleading results.

The `EXT:dbdoctor` pull request referenced in this context is:  
https://github.com/lolli42/dbdoctor/pull/171
