## Topic 1: Professional Engagement with Multilingual TYPO3 Projects

André shared that he will be working on a new multilingual project, which will involve language-country combinations. This project will bring him back into deeper involvement with translation handling in TYPO3. The setup may require two levels of language dimensions and could involve up to 30 combinations of regions and languages. Eric noted that this practical experience will be beneficial for the initiative’s goals.

## Topic 2: Handling Global and Local Content Mix

André described the project requirements for managing both global and local content. Scenarios include:
- Entire pages available only in specific regions.
- Global pages supplemented with localized content elements.
- Local pages that integrate selected global content.

Eric highlighted the need for clear fallback logic in such cases, especially when mixing content across localization scopes.

## Topic 3: Discussion on Fallback Logic and Technical Complexities

The team discussed technical implications of fallback mechanisms, particularly when using mixed-language content. André mentioned challenges related to routing and the need to ensure that editors can manage the setup. Mountpoints were mentioned as an option, but André expressed his dislike for them due to past negative experiences.

## Topic 4: Duplicate References and Language Synchronization Issues

Eric reported a severe bug encountered in a client project where over half a million duplicate image references were created in a single day due to copy operations across language setups. The duplication likely stemmed from:
- Copying content trees that referenced languages not available in the target site.
- Misuse of `Allow Language Synchronization` for image fields.
- Automated DeepL translations compounding the issue.

He suspects that default language fallbacks misinterpreted the unavailable languages, causing a recursive duplication. This led to system memory issues and a cluttered database.

## Topic 5: Existing and Missing Patches for Copy Processes

The team referenced an older patch that aimed to prevent the creation of orphaned records during copy operations. Although merged in version 13.0.4, this patch does not exist in TYPO3 v12. Eric suggested backporting the patch for his current project if an upgrade to v13 isn't feasible. The discussion included:
- Cherry-pick dates and related review history.
- Differences in behavior between v12 and v13 due to changes in the DataHandler and test infrastructure.
- Challenges in verifying proper localization filtering during copy operations.

## Topic 6: Testing Copy Behavior in TYPO3 v14

Eric conducted a live test using TYPO3 v14.1 and the new `theme-camino`. Together with André, they tested:
- Language configuration for two sites (English + German, French).
- Copying translated pages and checking for orphaned records.
- Use of `Allow Language Synchronization` in content and page settings.

The test confirmed that translations were incorrectly retained even when the target site did not support those languages. They identified resulting orphaned records in the database, confirming the bug.

## Topic 7: Plan to Develop New Patch for Page Translation Handling

Eric and André concluded that a new patch is needed to prevent orphaned `pages` records by aligning with the available languages in the `site configuration`. This would mirror the behavior already implemented for content elements. André noted an existing Forge ticket that exactly fits the issue and encouraged Eric to use it as a reference. Eric agreed to take ownership of the fix.

## Topic 8: Outlook for Patch Implementation

Eric stated his intent to work on the new patch, emphasizing its well-defined scope and potential value to the initiative. The team agreed to revisit progress during the next meeting and coordinate based on test outcomes and developer capacity.
